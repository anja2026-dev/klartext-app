Kapitel 1

Systemüberblick

Der KLARTEXT-Vertretungsassistent ist ein Backend-Modul, das bei Abwesenheit einer INGRA automatisch geeignete Vertretungen vorschlägt. Die Logik basiert auf Verfügbarkeit, Qualifikation, Einsatzort und Belastungszustand (Barometer).

Kernfunktionen:
Abwesenheitsmeldung erfassen · Verfügbare INGRAs filtern · Gewichteten Score berechnen · Priorisierte Vorschlagsliste an TK senden · Alle Vorgänge revisionssicher protokollieren

Beteiligte Akteure

AkteurRolleRechte
INGRASchulbegleitung – meldet eigene AbwesenheitEigene Daten lesen/schreiben, Barometer setzen
TK / KathrinTeamkoordination – wählt Vertretung ausAlle INGRAs sehen, Vorschläge bestätigen
AdminSystemverwaltung – Stammdaten pflegenVollzugriff auf alle Daten
SystemBackend – führt Filterlogik automatisch ausLesend auf alle, schreibend auf Vorschläge/Status

Kapitel 3

Vertretungslogik (Pseudocode)

Die Vertretungssuche läuft in 5 Stufen: 4 aufeinander aufbauende Filter, gefolgt von der Score-Berechnung zur Priorisierung.

Filter-Kette (Übersicht)

Alle INGRAs
→

→

→

→

3.1 Hauptfunktion

FUNCTION findVertretungen(abwesenheit_id):

  abwesenheit = DB.getAbwesenheit(abwesenheit_id)
  kandidaten  = DB.getAlleAktivenINGRAs()

  // STUFE 1: Zeitliche Verfügbarkeit prüfen
  kandidaten = filterVerfuegbarkeit(kandidaten, abwesenheit)

  // STUFE 2: Laufende Einsatzkonflikte ausschließen
  kandidaten = filterKonflikte(kandidaten, abwesenheit)

  // STUFE 3: Qualifikationsanforderungen des Kindes prüfen
  kandidaten = filterQualifikation(kandidaten, abwesenheit.kind_id)

  // STUFE 4: Barometer-Rot ausschließen
  kandidaten = filterBarometerRot(kandidaten)

  // STUFE 5: Score berechnen, sortieren, Vorschlagsliste bauen
  kandidaten = berechneScore(kandidaten, abwesenheit)
  kandidaten = sortByScore(kandidaten, DESC)
  vorschlaege = buildVorschlaege(kandidaten.top(5), abwesenheit)

  // STUFE 6: Speichern und TK benachrichtigen
  DB.saveVorschlaege(vorschlaege)
  notify.sendToTK(vorschlaege)

  RETURN vorschlaege

3.2 Filter 1 – Verfügbarkeit

FUNCTION filterVerfuegbarkeit(kandidaten, abwesenheit):

  result = []

  FOR EACH ingra IN kandidaten:
    zeitraum   = abwesenheit.von_datum BIS abwesenheit.bis_datum
    verfuegbar = true

    FOR EACH tag IN zeitraum:
      wochentag = getWochentag(tag)

      // Reguläre Verfügbarkeit prüfen
      regulaer = DB.getVerfuegbarkeit(ingra.id, wochentag)
      IF NOT regulaer OR NOT zeitraum.passt(regulaer):
        verfuegbar = false; BREAK

      // Ausnahmen prüfen (eigener Urlaub, Krankheit etc.)
      ausnahme = DB.getAusnahme(ingra.id, tag)
      IF ausnahme AND NOT ausnahme.verfuegbar:
        verfuegbar = false; BREAK

    IF verfuegbar:
      result.append(ingra)

  RETURN result

3.3 Filter 2 – Einsatzkonflikte

FUNCTION filterKonflikte(kandidaten, abwesenheit):

  result = []

  FOR EACH ingra IN kandidaten:
    konflikte = DB.getAktiveEinsaetze(ingra.id, abwesenheit.zeitraum)

    // INGRA nur wenn maximal 1 weiterer Einsatz im Zeitraum
    IF konflikte.count <= 1:
      ingra.belastung = konflikte.count   // 0 = frei · 1 = teilbelegt
      result.append(ingra)

  RETURN result

3.4 Filter 3 – Qualifikation

FUNCTION filterQualifikation(kandidaten, kind_id):

  kind        = DB.getKind(kind_id)
  pflichtQual = kind.benoetigte_qualifikationen   // z.B. ["Ersthelfer", "Sonderpaed"]
  mindestQual = kind.mindest_qualifikationen       // z.B. ["Erzieher"]

  result = []

  FOR EACH ingra IN kandidaten:
    hatPflicht = pflichtQual.all(q => ingra.qualifikationen.includes(q))
    hatMindest = mindestQual.any(q => ingra.qualifikationen.includes(q))

    IF hatPflicht AND hatMindest:
      ingra.qualScore = berechneQualScore(ingra.qualifikationen, pflichtQual)
      result.append(ingra)

  RETURN result

3.5 Filter 4 – Barometer Rot

FUNCTION filterBarometerRot(kandidaten):

  result = []
  heute  = Date.today()

  FOR EACH ingra IN kandidaten:
    barometer = DB.getBarometerAktuell(ingra.id, heute)

    IF barometer IS NULL:
      ingra.barometerFarbe = "unbekannt"    // neutral behandeln
      result.append(ingra)

    ELSE IF barometer.farbe != "rot":
      ingra.barometerFarbe = barometer.farbe
      result.append(ingra)

    ELSE:    // Rot → ausschließen, TK Warnung senden
      notify.sendBarometerWarnung(ingra, barometer)

  RETURN result

3.6 Score-Berechnung & Gewichtung

FUNCTION berechneScore(ingra, abwesenheit):

  score = 0

  // Kriterium 1: Gleiche Schule (höchste Priorität)
  IF ingra.hatEinsatzBei(abwesenheit.schule_id):
    score += 40

  // Kriterium 2: Entfernung Wohnort → Schule (Haversine)
  entfernung = geo.distanzKm(ingra.wohnort, schule.standort)
  IF entfernung <= 5:   score += 30
  ELIF entfernung <= 10: score += 20
  ELIF entfernung <= 20: score += 10
  ELSE:                   score += 0

  // Kriterium 3: Qualifikations-Match (0–20 Punkte)
  score += ingra.qualScore

  // Kriterium 4: Barometer
  IF   ingra.barometerFarbe == "gruen":     score += 10
  ELIF ingra.barometerFarbe == "gelb":      score += 5
  ELIF ingra.barometerFarbe == "unbekannt": score += 7

  // Kriterium 5: Geringe Belastung
  IF   ingra.belastung == 0: score += 10
  ELIF ingra.belastung == 1: score += 5

  RETURN score   // Maximal 110 Punkte

Score-Gewichtung auf einen Blick

40
Gleiche Schule
30
≤ 5 km Entfernung
20
Qualifikations-Match
10
Barometer Grün
10
Keine Zusatzbelastung
110
Maximum gesamt
Kapitel 4

Prozesskette

10 Schritte von der Abwesenheitsmeldung bis zur bestätigten Vertretung – inklusive Fallback bei keinem Ergebnis.

01
INGRA meldet Abwesenheit
INGRA öffnet App → Formular: Datum, Uhrzeit, Grund, Kind, Schule → Absendet → Status wird auf gemeldet gesetzt
02
Validierung der Eingabe
System prüft: Pflichtfelder vorhanden? Datum in Zukunft? Kind und Schule valide? → Bei Fehler: Fehlermeldung an INGRA, Abbruch
03
Abwesenheit wird gespeichert
DB.insertAbwesenheit() → ID wird generiert → Status: gemeldet → Timestamp gesetzt
04
Trigger: Vertretungssuche starten
Event: abwesenheit.created → Queue: vertretungssuche.run(abwesenheit_id) → Asynchrone Verarbeitung
05
Filterlogik läuft durch (4 Stufen)
findVertretungen() führt alle 4 Filter durch → Verfügbarkeit → Konflikte → Qualifikation → Barometer
06
Score wird berechnet
berechneScore() für jeden Kandidaten → Sortierung absteigend nach Gesamtscore (max. 110 Punkte)
07
Vorschlagsliste wird erstellt
Top 5 Kandidaten → buildVorschlaege() → Datensätze mit Score, Entfernung, Qualifikation und Begründungstext
08
TK wird benachrichtigt
Push-Notification + E-Mail an TK/Kathrin → Übersicht mit allen Vorschlägen, Score, Begründung und Kontaktdaten
09
TK wählt Vertretung aus
TK bestätigt einen Vorschlag → Status: vertretung_gefunden → Gewählte INGRA erhält Push-Notification und E-Mail
10
Fallback: Kein Ergebnis nach Filterung
0 Kandidaten vorhanden → TK erhält sofortige Alarmierung → Status: offen → Manuelle Suche erforderlich
